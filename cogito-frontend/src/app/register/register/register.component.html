SEEance - Terms of Service
Purpose

SEEance is a prototypical software repository analysis platform and framework developed by Felix Riedl at the University of Regensburg (2020).

It is currently hosted for public evaluation reasons at the University of Regensburg, Germany.

The purpose of SEEance is to provide educators in the field of Software Engineering with an uncomplicated, extensible toolkit for inspecting their student groups' development processes.

The project's current stage is a functioning prototype, hence delivered as is. Use it at your own peril.


What SEEance can do
provide easy access to typical VCS analysis metrics as described in related work give you hints on how to identify typical anomalies in your projects' analyses give an impulse to inspect supposed anomalies together with your students
What SEEance cannot do
interpret the analysis results for you lie a sufficient and reliable foundation for grading replace direct communication with your students
Which data we use (and why)
Used Data from Github

You authenticate for SEEance via Github, which may authorize SEEance to access private information tied to your account. This includes:

public account information (username, profile picture etc.) names for public and private repositories cloning/pulling public and private repositories via your account to our server accessing metadata tied to public and private repositories via your account (issues, pull requests etc.)
Security Measures and Caveats

We use this access carefully, so we

will never cache access tokens, passwords etc. pull data beyond your input (e.g. will only access repositories, that you have actually selected for analysis) only authorize for read access

Albeit we are striving for an adequate degree of application security, we cannot take responsibility for any damage caused by our application. We strongly recommend following a few security guidelines:

never add sensible repositories to analyses (e.g. repositories containing sensible data such as personal information or keys, passwords) only grant access to organizations, that are absolutely necessary if in doubt, we propose using a burner account for testing
Collection of interaction data

Also, we are collecting interaction data while on the page for evaluation reasons

dwell time on application pages page-internal keyboard events (number per page) page-internal click events (target) page-internal navigation events
Purpose of data collection

We are caching data and repository clones for analysis reasons on a server at the University of Regensburg.

data collected via this analysis will be exclusively used for scientific purposes (masters thesis of Felix Riedl as well as follow-up publications) we will never publish repository analysis caches produced by SEEance and wipe all those caches after concluding our evaluation phase we will never publish personal information or information, that may be retraced to individual platform users
